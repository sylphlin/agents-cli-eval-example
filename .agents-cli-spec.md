# Agent Spec: 對聯機器人 (Couplet Generator)

## Overview
對聯機器人是一個基於 Google ADK（Agent Development Kit）與大型語言模型（LLM）建構的傳統對聯創作代理。當用戶給出創作題目與具體條件（如主題節日、字數限制、嵌字要求、平仄對仗規範等），代理會自動生成符合條件的工整上聯與下聯。

本專案作為 `agents-cli eval` 的標準測試與評估範例，展示如何對 AI 代理進行「硬性邏輯條件驗證」與「軟性文學意境評估」。

## Implementation Metadata
- **專案名稱**：`agents-cli-eval-example`
- **核心框架**：`google-adk` (`agents-cli`)
- **Agent 名稱**：`couplet_generator` (App 容器名稱：`app`)
- **使用模型**：`gemini-flash-latest` (設定 `attempts=3` 重試機制)
- **佈署與開發模式**：`--prototype`（純本機開發與 Eval 測試驗證）

## Core System Instructions & Rules
在 `app/agent.py` 中，代理必須嚴格遵循以下創作鐵律：
1. **字數絕對精準（首要硬性條件）**：
   - 題目若要求 N 字對聯，上聯與下聯的漢字數必須各自精準等於 N 個漢字（標點符號不計入，絕切勿多寫或少寫任何一字）。
2. **六大嵌字格律落點精準控制**：
   - **鶴頂格（冠頭格）**：指定二字分別為「上聯第1字」與「下聯第1字」。
   - **雁足格（藏尾格）**：指定二字分別為「上聯最後一字」與「下聯最後一字」。
   - **腹嵌格**：指定二字分別位於上下聯的正中央字位。
   - **魁鬥格**：上聯第1字嵌第一個字，下聯最後一字嵌第二個字。
   - **連理格**：指定二字連續相鄰出現在語句中。
   - **碎錦格**：指定多字分散全數出現在上下聯中。
3. **對仗工整雅致**：詞性相對、意境喜慶優雅。
4. **標準輸出格式規範**：
   ```text
   上聯：[剛好N個漢字]
   下聯：[剛好N個漢字]
   ```

## Example Use Cases
- **輸入**：「請創作一副春節賀歲對聯，要求上下聯各7個字，上聯需嵌入『春』字，下聯需嵌入『福』字。」
  **輸出**：
  上聯：迎春接福萬象新
  下聯：納福迎祥百業興
- **輸入**：「寫一副慶祝科技公司開工的5字對聯。」
  **輸出**：
  上聯：雲端開駿業
  下聯：智慧展宏圖
